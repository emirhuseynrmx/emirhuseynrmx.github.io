Problem neydi? What was the problem?

📉

Backtest kazanıyor, canlı kaybediyorBacktest wins, live trading loses

Klasik overfitting sorunu değil bu. Strateji gerçekten çalışıyor — ama belirli piyasa koşullarında başarısız oluyor. Yüksek volatilite dönemlerinde, counter-trend girimlerde, anormal hacim sırasında. Strategy Tester bu koşulları ortalamayla yutuyor. Not classic overfitting. The strategy genuinely works — but fails under specific market conditions. During high volatility, counter-trend entries, abnormal volume. Strategy Tester averages over these conditions and hides them.

🔍

Problem tipi: ikili sınıflandırmaProblem type: binary classification

Hedef değişken: bad_trade = 1 eğer trade_return < -1%, aksi halde 0. Her işlemi ikiye ayırıyoruz. Hangi özellikler kötü işlemleri ayırt ediyor? ATR (volatilite), volume z-score (hacim anomalisi), EMA trend yönü — bunlar girdi değişkenleri. Target variable: bad_trade = 1 if trade_return < -1%, else 0. Binary split on every trade. Which features distinguish bad trades? ATR (volatility), volume z-score (volume anomaly), EMA trend direction — these are the features.

🧠

Sadece skor yetmez: aksiyon kuralı lazımScore alone isn't enough: need an entry rule

"Bu işlem yüksek riskli" demek tek başına yetersiz. Trader'ın ihtiyacı: "Bu koşullar olsaydı girilebilirdi." DiCE ile minimum-değişiklikli counterfactual: hangi feature değeri değişse risk eşiğinin altına iner? Bu bir filtre kuralına dönüşür. "This trade is high risk" alone is insufficient. Trader needs: "These conditions would have made it safe." DiCE minimum-change counterfactual: which feature value change would push risk below threshold? This becomes a filter rule.

Neden XGBoost? sklearn GradientBoosting veya Random Forest değil mi? Why XGBoost? Not sklearn GradientBoosting or Random Forest?

Random Forest İyi baseline ama sklearn'deki SHAP implementasyonu RandomForest için 3 boyutlu array döndürür: (n_samples, n_features, n_classes). Bu şekli yanlış handle eden kod sessizce çöküyor. XGBoost'ta bu sorun yok. Ayrıca RF missing value'ları native desteklemiyor.Good baseline but sklearn's SHAP for RandomForest returns 3D array: (n_samples, n_features, n_classes). Code that handles this shape wrong silently breaks. XGBoost doesn't have this issue. Also RF doesn't natively handle missing values.
sklearn GBM Küçük dataset'te iyi çalışır. 1500 işlem için yeterli olurdu. Ama XGBoost'un L1+L2 regularizasyonu (alpha + lambda parametreleri) over-dense tree'lere karşı daha iyi korur. Ayrıca XGBoost'un histogram tabanlı bölmesi %40 daha hızlı.Works fine on small datasets. Would be sufficient for 1500 trades. But XGBoost's L1+L2 regularization (alpha + lambda) provides better protection against over-dense trees. Also XGBoost's histogram splits are ~40% faster.
XGBoost Native NaN desteği (market open'da volume_zscore tanımsız olabilir). L1+L2 regularizasyon birleşik. SHAP TreeExplainer tam uyumlu — shape sorunu yok. feature_types parametresi ile kategorikal (symbol) doğrudan geçilebilir. Seçim nedeni: hem performans hem araç ekosistemi.Native NaN support (volume_zscore may be undefined at market open). Combined L1+L2 regularization. SHAP TreeExplainer fully compatible — no shape issues. feature_types handles categoricals (symbol) directly. Selection reason: both performance and tooling ecosystem.

Veri doğrulama ve feature engineering — ham CSV'den modele Data validation and feature engineering — from raw CSV to model

Ham trades.csv'yi XGBoost'a doğrudan atmazsın. Önce verinin ne durumda olduğunu anlarsın, sonra eksikleri doldurursun, sonra ham fiyat/hacim verisini anlamlı sinyallere dönüştürürsün. Bu adımlar modelin sonucundan daha çok zaman alır. You don't feed raw trades.csv directly to XGBoost. First you understand what state the data is in, then fill gaps, then transform raw price/volume into meaningful signals. These steps take more time than the model itself.

ADIM 2 — NULL HANDLING (ne doldurulur, ne atılır?)
# volume_zscore: ilk 20 günde moving avg yok → NaN
# Strateji: bu satırları silme, XGBoost native NaN destekler
df["volume_zscore"] = df["volume_zscore"].fillna(0.0) # sadece raporlama için
# atr_pct: split day'de volume=0 → ATR tanımsız → ffill
df["atr_pct"] = df["atr_pct"].ffill() # önceki değeri taşı
# trade_return: hedef değişken — NaN ise o satır dışarıda bırak
df = df.dropna(subset=["trade_return"]) # target null olamaz
ADIM 3 — FEATURE ENGINEERING (ham fiyat → sinyal)
# ATR%: volatilite normalize edilmiş — $5 hisse ile $500 hisse karşılaştırılabilir
high_low = df["High"] - df["Low"]
true_range = high_low.rolling(14).mean()
atr_pct = true_range / df["Close"] * 100
# Volume z-score: bugünün hacmi normalden ne kadar farklı?
vol_mean = df["Volume"].rolling(20).mean()
vol_std = df["Volume"].rolling(20).std()
volume_zscore = (df["Volume"] - vol_mean) / vol_std
# EMA trend: kısa/uzun vade trend yönü farkı
ema20 = df["Close"].ewm(span=20).mean()
ema50 = df["Close"].ewm(span=50).mean()
ema_trend = (ema20 - ema50) / df["Close"] # normalize
