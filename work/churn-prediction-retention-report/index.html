Neden bu yaklaşım?Why this approach?

  • Model teslim değil: müşteri risk dosyası + aksiyon kuyruğu + sınırları açıklayan rapor teslim edildi. Siyah kutu model değil, anlaşılabilir çıktı.Model is not the delivery: client receives a risk file + action queue + a report explaining the limits. Not a black-box model — understandable output.
  • Feature engineering sektöre özgü: SaaS için login sıklığı, feature kullanımı, destek talebi patikası. Generic çözüm yerine iş bağlamı entegre edildi.Feature engineering is domain-specific: login frequency, feature usage, support ticket patterns for SaaS. Business context integrated instead of a generic solution.
  • Holdout validasyonu zorunlu: eğitim setinde iyi görünen model değil, görülmemiş veriye genelleşen model. Overfitting riski açıkça raporlandı.Holdout validation mandatory: not a model that looks good on training data, but one that generalizes to unseen data. Overfitting risk explicitly reported.
  • SHAP değerleri dahil: hangi feature'ın hangi müşteri için en çok etki ettiği açıklanabilir. CS ekibi "neden?" sorusunu cevaplayabilir.SHAP values included: which feature drives which customer's risk is explainable. CS team can answer "why?" for each churn prediction.
