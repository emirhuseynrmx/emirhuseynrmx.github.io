Ne yapar? What it does

📋

watchlist.json

Her ürün için: name, URL, css price_selector ve target_price. JSON dosyasına yeni satır ekle, pipeline bir sonraki çalıştırmada onu da izlemeye alır. Kod değişmez. Per product: name, URL, css price_selector, and target_price. Add a new entry to the JSON, pipeline picks it up next run. No code change.

🌍

Çift locale fiyat parseDual-locale price parsing

Avrupa formatı: 1.299,99 (nokta binlik, virgül ondalık). US formatı: 1,299.99 (virgül binlik, nokta ondalık). İki formatı da otomatik tanır ve float'a çevirir. European format: 1.299,99 (dot thousands, comma decimal). US format: 1,299.99 (comma thousands, dot decimal). Auto-detects both and converts to float.

🚨

Hedef fiyat uyarısıTarget price alert

Fiyat ≤ target_price olduğunda uyarı tetiklenir. delta = target_price − price olarak hesaplanır ve rapora yazılır. Her ürün için ayrı eşik. Alert triggers when price ≤ target_price. delta = target_price − price is computed and written to report. Separate threshold per product.

⏱️

HTTP timeout=15s

httpx async client ile timeout=15 saniye. Cevap vermeyen siteler pipeline'ı askıda bırakmaz. Timeout olan URL'ler manifest'e hata olarak kaydedilir. httpx async client with timeout=15 seconds. Unresponsive sites don't stall the pipeline. Timed-out URLs logged as errors in the manifest.

📁

CSV snapshot + JSON manifestCSV snapshot + JSON manifest

Her çalıştırma: timestamp'li CSV snapshot (tüm mevcut fiyatlar) ve JSON manifest (schema fingerprint: SHA-256 ilk 16 karakter, kaç URL kontrol edildi, kaç uyarı tetiklendi). Each run: timestamped CSV snapshot (all current prices) and JSON manifest (schema fingerprint: SHA-256 first 16 chars, URLs checked, alerts triggered).

🔍

CSS selector tabanlıCSS selector-based

API anahtarı yok, scraper aboneliği yok — sadece tarayıcıda inspect et, fiyat elementinin CSS selector'ını al, watchlist'e ekle. BeautifulSoup ile güvenilir parse. No API key, no scraper subscription — just inspect in browser, grab the CSS selector of the price element, add to watchlist. Reliable parsing via BeautifulSoup.

watchlist.json okundu, sayfalar çekildiwatchlist.json read, pages fetched

watchlist.json'dan 3 ürün yüklendi: her birinde name, URL, css price_selector, target_price. httpx async client (timeout=15s) ile 3 URL paralel olarak çekildi. BeautifulSoup her sayfada price_selector'ı aradı. İki ürün Avrupa formatı (1.299,99), biri US formatı (1,299.99) — her ikisi de parse edildi. 3 products loaded from watchlist.json: each with name, URL, css price_selector, target_price. httpx async client (timeout=15s) fetched all 3 URLs in parallel. BeautifulSoup searched for price_selector on each page. Two products European format (1.299,99), one US format (1,299.99) — both parsed correctly.

02

Uyarı tetiklendi: fiyat ≤ hedefAlert triggered: price ≤ target

Ürün A: çekilen fiyat 249,99 €, target_price=259,99 €. Koşul sağlandı: fiyat ≤ hedef. delta = 259,99 − 249,99 = 10,00 € hesaplandı. Uyarı kaydı oluşturuldu. Ürün B ve C: fiyat hedef üstünde, uyarı yok. Product A: fetched price 249.99 €, target_price=259.99 €. Condition met: price ≤ target. delta = 259.99 − 249.99 = 10.00 € computed. Alert record created. Products B and C: price above target, no alert.

Neden bu yaklaşım?Why this approach?

  • Tekrarlanan kontrol, tekrarlanan zafer: saatlik cron, her çalıştırma önceki fiyatla karşılaştırma, sadece değişen ürünlerde alarm. Gürültü yok.Repeatable check, repeatable win: hourly cron, each run compares against last price, alert only on changed products. No noise.
