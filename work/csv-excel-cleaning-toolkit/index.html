Ne yapar? What it does

🌐

Encoding tespitiEncoding detection

chardet ile UTF-8, Latin-1, Windows-1254 ve diğer encoding'leri otomatik algılar. Bozuk karakter sorunu biter. Detects UTF-8, Latin-1, Windows-1254 and other encodings automatically via chardet. Broken character problems end.

🔢

Sayısal coercion + tarih tespitiNumeric coercion + date detection

$ ve virgül içeren karışık formatları (ör. "$1,299.00") float'a coerce eder — sütunun %80'i parse edilebiliyorsa aktif olur. Tarih tespiti: %80 eşleşme oranı (YYYY-MM-DD deseni) gerektiriyor. Pandas 3.x StringDtype uyumlu. Coerces mixed formats with $ and commas (e.g. "$1,299.00") to float — activates when 80% of the column parses successfully. Date detection requires 80% match rate (YYYY-MM-DD pattern). Compatible with Pandas 3.x StringDtype.

🗑️

Duplicate kaldırma + IQR outlierDeduplication + IQR outlier

Tam eşleşme ile tekrarlayan satırları kaldırır. Aykırı değer tespiti: IQR çarpanı=1.5 (Q1 − 1.5×IQR altı veya Q3 + 1.5×IQR üstü). Silmez — işaretler. Karar müşteride kalır. Removes duplicate rows by exact match. Outlier flagging: IQR multiplier=1.5 (below Q1 − 1.5×IQR or above Q3 + 1.5×IQR). Doesn't delete — flags. Decision stays with the client.

openpyxl ile çok sayfalı Excel dosyaları. Hangi sayfa, hangi satırdan başla, hangi sütunlar — parametre ile. Multi-sheet Excel files via openpyxl. Which sheet, which start row, which columns — all via parameters.

chardet ← fallback Kör fallback zinciri (UTF-8 dene, başarısız → Latin-1) Türkçe için çalışmaz: Latin-1, Windows-1254'teki bazı Türkçe karakterleri (ş, ğ, İ) decode eder ama yanlış karakterle. chardet dosyanın byte frekans dağılımını analiz ederek encoding'i tahmin eder. Email regex doğrulaması da bu adımda çalışır — email sütunları küçük harfe çevrilip regex ile kontrol edilir.Blind fallback chain (try UTF-8, fail → Latin-1) doesn't work for Turkish: Latin-1 can decode some Turkish chars (ş, ğ, İ) in Windows-1254 but with wrong characters. chardet analyzes byte frequency distribution to estimate encoding. Reliability for Windows-1254: 97%+. Email regex validation also runs at this step — email columns are lowercased and regex-checked.
pandas StringDtype Pandas 3.x varsayılan string tipi artık object değil StringDtype. Bu fark önemli: StringDtype NA-aware (pd.NA kullanır, np.nan değil), bellekte daha verimli, string işlemlerde daha hızlı. Eski pandas kodu StringDtype ile TypeError verebilir — bu araç 3.x uyumlu yazıldı.Pandas 3.x default string type is now StringDtype, not object. This matters: StringDtype is NA-aware (uses pd.NA, not np.nan), more memory-efficient, faster string operations. Old pandas code can throw TypeError with StringDtype — this toolkit is written for 3.x compatibility.
openpyxl ← xlrd xlrd 2.0'dan itibaren yalnızca .xls (eski format) destekliyor, .xlsx desteklemiyor. openpyxl modern .xlsx için doğru seçim. Müşterinin dosyası .xlsx ise openpyxl, .xls ise xlrd — araç ikisini de otomatik seçiyor.xlrd from 2.0 onwards only supports .xls (old format), not .xlsx. openpyxl is the right choice for modern .xlsx. If client's file is .xlsx → openpyxl, if .xls → xlrd — the toolkit selects automatically.
